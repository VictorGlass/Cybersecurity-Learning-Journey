# Fundamentals

## Overview


Network security fundamentals are based on understanding how devices communicate across network and how protocols operate at different layers.

The TCP/IP and OSI models help organize network communication into multiple layers, each with specific responsibilities and protocols.


# TCP/IP and OSI Models

Different protocols operate at different layers of the networking models.


## Application Layer

Protocols used for communication between applications and services.

Examples:
- FTP.
- DNS.
- HTTP.
- SSH.
- SMTP.


## Transport Layer

Responsible for end-to-end communication and data transport.

Protocols:
- TCP.
- UDP.

## Network Layer

Responsible for routing and addressing between networks.

Protocols:
- IP.
- IPsec.
- ICMP.


## Access Layer

Responsible for local network communication and hardware-level addressing.

Protocols:
- MAC.
- ARP.


# Encapsulation

During encapsulation, each protocol adds its own header to the packet as the data moves through the network layers.

When the destination device receives the packet, it performs the reverse process called decapsulation, processing and removing the headers layer by layer.



# Importance

Understanding encapsulation and network layers is essential for:
- Troubleshooting network issues.
- Packet analysis.
- Network monitoring.
- Cybersecurity investigations.
- Threat detection.


# Personal Insight

Learning how protocols interact across different network layers helps build a stronger foundation for both defensive and offensive cybersecurity practices.
